Q: What is a good name? A: Ask Uncle Bob!
Use intention-revealing names 
            int d; //elapsed time in days
        
            int elapsedTimeInDays;
Avoid disinformation
  • AccountList
  • AccountGroup
  • Accounts
Make meaningful distinctions
  • a1, a2, ..., aN
  • Product vs. ProductInfo vs. ProductData
Use pronounceable names 
            DateTime genymdhms;
        
            DateTime generationTimestamp;
Use searchable names
  • 7 vs. MAX_CLASSES_PER_STUDENT
Avoid Encoding
  • m_ prefix
Don't be cute
  • HolyHandGrenade
  • DeleteItems
Pick one word per concept
  • xxController, xxManager, xxDriver
  • xxBuilder, xxCreator
Use problem domain names
Use solution domain names
  • .NET naming guidance http://msdn.microsoft.com/en-us/library/ms229002(v=vs.110).aspx
  • ASP.NET MVC/Web API naming convention
  • Design patterns
And finally...

Keep refactoring bad names!
